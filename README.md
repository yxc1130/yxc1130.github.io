## 游戏简介

在一片被废铁与污染覆盖的废土上，矗立着一座神秘的高塔。玩家将扮演一名废土拾荒者，选择六大职业之一，通过爬塔战斗、收集废料、合成装备、触发事件，一步步登上塔顶。

### 核心玩法

- **爬塔地图**：三路分支路线，每层包含战斗、精英、商店、工匠、事件、篝火等节点
- **回合制卡牌战斗**：消耗能量打出手牌，包含攻击、防御、技能、道具等卡牌
- **废料合成系统**：收集废料合成道具卡，道具卡可升级（1★→2★→3★→4★）
- **职业系统**：6个职业各有独特机制
- **难度选择**：简单/普通/困难，影响怪物属性、奖励和初始资源
- **Boss战**：每层末尾有Boss战

### 六大职业

| 职业 | 英文名 | 特色 | 风格 |
|------|--------|------|------|
| 🔧 拆解师 | The Wrecker | 瓦解敌人增益，驱散增伤 | 破甲爆发 |
| ⚒️ 锻造师 | The Forger | 消耗废料强化卡牌 | 卡牌强化 |
| ♻️ 回收师 | The Recycler | 弃牌触发回收循环 | 无限循环 |
| 🧪 毒药师 | The Toxic | 毒素叠加，死亡传染 | 群体传染 |
| ⚡ 过载师 | The Overload | 油污过载，极限爆发 | 极限爆发 |
| 🧩 组装师 | The Assembler | 多牌合体为组合卡 | 策略组合 |

## 技术实现

- **纯前端**：单 HTML 文件，零依赖
- **Web Audio API**：程序化生成 BGM（废土民谣风格芯片音乐）
- **Canvas 2D**：实时生成角色精灵和怪物精灵
- **GitHub Pages**：免费永久部署

## 对话复盘 & 效率评估

本项目的开发过程中，用户与 AI 助手进行了多轮对话推进开发。以下是对整个对话的复盘总结：

### 对话脉络

| 阶段 | 轮次 | 内容 | 推进方式 |
|------|------|------|----------|
| 1 | 1-3 | 初始需求：从已有游戏想法出发，AI 直接创作完整游戏代码 | AI 全量生成 |
| 2 | 4-6 | 部署上线、修复商店 Bug、改进合成台、生成音频文档 | 用户反馈 → AI 修复 |
| 3 | 7-9 | BGM 从氛围噪音 → 芯片音乐 → 废土民谣风格 3 次迭代 | 用户不满意 → AI 调整 |
| 4 | 10-12 | 角色精灵从无 → 像素 → 大头 → 剪影 → 死亡细胞风格 | 用户不满意 → AI 多次重做 |
| 5 | 13-14 | 英文名 + 呼吸动画 + 职业攻击动画 | 用户提需求 → AI 实现 |
| 6 | 15-17 | 部署从 Netlify（额度用尽）→ localtunnel（临时）→ GitHub Pages（永久） | 问题排查 → 方案切换 |
| 7 | 18 | 数值平衡调整、README 总结 | 综合收尾 |

### 项目推进效率评价

**综合评分：⭐⭐⭐⭐（4/5）**

| 维度 | 评价 | 说明 |
|------|------|------|
| **响应速度** | ⭐⭐⭐⭐⭐ | AI 每次收到反馈后即时修改并部署 |
| **需求理解** | ⭐⭐⭐⭐ | 大部分需求一次理解到位，偶有偏差需多轮修正 |
| **代码质量** | ⭐⭐⭐⭐ | 结构清晰、注释完整，单文件架构维护简单 |
| **问题修复** | ⭐⭐⭐⭐⭐ | 商店 Bug、API 失效、部署失效等问题均及时解决 |
| **审美迭代** | ⭐⭐⭐ | 视觉/BGM 风格需要多次试错才能接近用户预期 |
| **部署运维** | ⭐⭐⭐⭐ | 从 Netlify → 临时隧道 → GitHub Pages 最终方案稳定 |

### 用户角色评价

用户的推进方式非常高效：
- **需求明确**：每个需求点清晰具体（"全身像"、"呼吸抖动"、"英文名"）
- **及时验收**：每轮修改后立即上线测试，快速反馈"行/不行"
- **接受度好**：给 AI 试错空间（BGM 风格尝试了 3 种才定稿）
- **坚决不将就**：不满意的结果（形象不行、部署没更新）直接指出，推动 AI 必须解决

这种"提需求 → 验收 → 反馈"的闭环是项目快速推进的关键。

### 关键数据

- **总对话轮次**：约 18 轮
- **部署迁移次数**：3 次（Netlify → localtunnel → GitHub Pages）
- **BGM 大改次数**：3 次（氛围 → 芯片 → 民谣）
- **精灵风格大改次数**：2 次（像素 → 死亡细胞）
- **从零到可玩**：第 1 轮对话即生成可运行的完整游戏
- **从可玩到精良**：后续 17 轮围绕体验打磨
